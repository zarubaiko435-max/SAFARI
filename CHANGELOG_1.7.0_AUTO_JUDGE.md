# CHANGELOG — SAFARI 1.7.0 AUTO JUDGE

## Головна зміна

Повністю замінено неправильний для основного трейдингу цикл «відкрий сесію → надсилай багато скрінів → дозбирай поля» на режим:

```text
ТИКЕР + CALL/PUT + приблизний страйк → автоматичне дослідження → готове рішення
```

## Додано

- Парсер однорядкових команд:
  - `SOFI CALL 16.5`
  - `SOFI PUT 16,5`
  - `SOFI CALL-PUT 16.5`
  - `ТРЕЙДИНГ SOFI CALL СТРАЙК 16,5`
- Автоматичний read-only market research через Webull OpenAPI.
- Пошук контракту біля приблизного страйку та орієнтовно 28 DTE.
- Збір і нормалізацію Bid/Ask, spread, OI, Volume, IV і Greeks.
- Збір 5m/15m bars і розрахунок EMA9, EMA21, RSI14, VWAP, ATR14.
- Збір earnings, analyst target і rating.
- Актуальні новини й каталізатори через OpenAI web search.
- Детерміноване порівняння CALL проти PUT.
- Повний аудит через `ЧОМУ?` з аргументами за/проти та джерелами.
- Новий модуль `safari_autojudge.py` без Telegram, OpenAI чи broker imports.
- 12 регресійних тестів Auto Judge.

## Правила рішення

- Критично відсутні live-дані → `WAIT`, а не вигаданий `TAKE`.
- Сильна технічна суперечність напрямку → `PASS`.
- Неприйнятний spread, ліквідність, строк або Greeks → hard stop/ризик.
- Earnings до 3 днів → `WAIT`.
- `TAKE` можливий лише за достатньої сили, прийнятного ризику й підтвердженого напрямку.
- Для `CALL-PUT` оцінюються обидва боки; показується сильніший результат і повне порівняння в `ЧОМУ?`.

## Безпека

- `READ_ONLY_MODE = True`.
- У Webull adapter немає order placement, modification, replacement або cancellation methods.
- Немає автоматичних повторів після Webull rate limit.
- Помилки джерел не маскуються вигаданими даними.

## Сумісність

- Старі `WEBULL`, `GUARDIAN`, `МОЇ ПОЗИЦІЇ`, `ДОСЬЄ`, `VISION` та пам’ять залишені.
- Screenshot flow більше не є основним шляхом для нової торгової ідеї.
